# TDD bootstrap and CI for CDK sleep audio pipeline

This change replaces the CDK scaffold's inert test stub with a working pytest setup, adds a GitHub Actions CI workflow that runs tests and synthesizes the stack, and drops in an architecture doc describing the intended event-driven pipeline. The approach is straightforward: shared fixtures in conftest, two lightweight assertions on the empty stack, and a single-job CI pipeline covering pytest → cdk synth → cdk diff.

**Watch for:** CI will fail on `cdk synth` because no AWS credentials or account/region context are configured in the workflow (confirmed). The `pytest-cov` dependency is declared but never invoked — no coverage reporting exists in CI or pytest config (confirmed).

## High-level view

The test infrastructure moves from a commented-out stub to a conftest-driven fixture pattern that synthesizes the stack once per test.

The CI workflow installs both Python and Node runtimes (CDK needs jsii), runs pytest, then attempts `cdk synth` and `cdk diff`. The synth step requires CDK to resolve an AWS account and region, but neither is provided via environment or `cdk.json` context, so it will fail in CI without additional configuration. The `cdk diff` step has `continue-on-error: true` (it compares against a deployed stack that doesn't exist), but `cdk synth` does not, and it has the same prerequisite problem.

ARCHITECTURE.md is aspirational documentation — it diagrams S3, Lambda, EventBridge, and Step Functions that don't exist yet as a roadmap artifact.

## CI synthesis without AWS context

`cdk synth` resolves the target account and region from the environment (`CDK_DEFAULT_ACCOUNT`, `CDK_DEFAULT_REGION`) or from explicit `env` on the Stack. The workflow provides neither. Running in GitHub Actions with no AWS credentials means CDK will emit an "Unable to resolve AWS account" error and the job will fail. The `cdk diff` step already anticipates failure with `continue-on-error: true`, but `cdk synth` does not — yet it has the same prerequisite problem. The fix is either to supply dummy account/region context for synthesis (sufficient for template generation) or to mark the step as non-blocking.

## Unused coverage dependency

`pytest-cov` is added to `requirements-dev.txt` but neither `pytest.ini` nor the CI step invokes it. The `pytest.ini` file contains only `testpaths = tests` with no `addopts = --cov`. This means coverage data is never collected.

## NODE_OPTIONS concern

The FEAT-001 spec mentions clearing `NODE_OPTIONS` to prevent jsii breakage. The CI workflow doesn't include `env: { NODE_OPTIONS: "" }`. On most ubuntu-latest runners this is a non-issue today, but it's a fragile assumption — GitHub periodically changes runner images and environment variables. Adding the explicit override costs nothing and matches the project's own documentation.
