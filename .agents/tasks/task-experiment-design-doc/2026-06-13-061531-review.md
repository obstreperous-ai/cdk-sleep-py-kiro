A new `EXPERIMENT.md` captures the full experimental methodology behind this repository: goals, actor setup, prompting patterns, issue history, key decisions, and preliminary observations. The README documentation table gets a corresponding link. No code or tests were modified—this is a documentation-only addition.

**Watch for:** The key decisions table is largely duplicated from SUMMARY.md with minor wording changes (confirmed). The issue history table references PR associations that a reader cannot verify from this repo alone—issue/PR numbers are stated but not linked as clickable URLs (confirmed). The experiment matrix in the "Actors & Setup" section is mostly placeholder cells, which may confuse readers about whether the grid is real or aspirational (likely).

## High-level view

EXPERIMENT.md is a 202-line document that formalizes what the repo represents within the broader "5 languages x 3 AIs" study. It covers methodology (TDD, issue-driven, architecture-as-code), identifies Kiro by Amazon as the sole AI actor with Python/CDK as the language flavor, and summarizes all 14 development issues in a table with PR references.

The preliminary observations section is the most original content and offers genuine reflections (state machine testing pain, NODE_OPTIONS quirk, Polly integration approach) that would be useful for cross-cell comparison in the experiment.

## Content overlap with SUMMARY.md

The key decisions table in EXPERIMENT.md (lines 148–160) reproduces the same 10 decisions found in SUMMARY.md with near-identical "Rationale" text. The added "Trade-off" column is the only net-new information. For a standalone experiment design document this is defensible—evaluators shouldn't need to cross-reference multiple files—but it creates a maintenance burden. If the decisions evolve, two files need updating. The document already links SUMMARY.md in its References section, so a reader can find the canonical list there.

## Issue history navigation

Issue and PR numbers are written as plain text (`#1`, `PR #2`) rather than as Markdown links (`[#1](https://github.com/obstreperous-ai/cdk-sleep-py-kiro/issues/1)`). On GitHub.com these will auto-link via shorthand notation, but anyone reading the document outside GitHub (rendered docs, exported PDF) gets no navigation affordance. Given that the document's audience includes evaluators who may be comparing across repositories, clickable URLs would serve better.

## Experiment matrix ambiguity

The 5x3 grid marks this repo's position but fills every other cell with `...`. The section doesn't explain whether the other cells exist, are planned, or are tracked elsewhere. A reader encountering this document in isolation can't determine whether the experiment is fully staffed or aspirational. A single sentence clarifying the status ("Other cells are tracked in [location]" or "This matrix is the target design; not all cells are populated") would resolve the ambiguity.
